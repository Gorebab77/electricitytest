[규격] - 기능요구사항


3. 기능 요구사항
3.1 주요 동작 요구사항
✅주요 동작요구 사항
@계기 기본 동작
  • 배터리모드 : 배터리를 기반으로 동작하는 모드
  • 무전압검침 : 육안으로 확인할수 있도록 표시항목
    1회순환 후 종료
  • 무전압설정 : 무선통신을 이용하여 계기의 계량,
    계측정보 수집 및 기능 설정하는 모드이며 별도의
    세션종료(DISC) 명령이 없는 경우 5분 후 종료.
  • 전원모드 : 사용전원이 인가되어 현장 또는 원격
    으로 계량,계측 정보 수집 및 기능 설정
@계측 기능
  • 각상별 전압,전류,전압전류의 위상각, 전압THD
    및 주파수의 순시 계측
@계량 기능
  •  계량 항목 : 유효 및 무효전력량, 최대수요전력
  • 확정 주기(저장)
  • 비정기 : 비정기적으로 계량 항목 확정
    • 원격으로 특정 일시에 검침 실시
    • 현장에서는 버튼을 이용하여 수동으로
      검침실시
    • 프로그램입력, 프로그램예약적용,
      계량모드 변경시 실시
  • 미래로 시간을 변경 또는 정·복전 시
    정기/비정기 검침일을 지날경우
  • 계절변경 : 계절변경시 계량항목 확정
  • 시간을 변경하거나 정·복전 시 계절변경이
    발생했을 경우
  • 확정벙법 : 전력량 확정(SR), 최대수요전력
    확정(DR)
  • 설정종류 : SR·DR,SR
  • 저장 방법 및 주기
  • 정기(연속 6개월), 비정기(4개월),
    계절변경(4회분) 확정데이터 별도구분 저장
@부하 이력 (Load Profile : LP)
  • 항목 : 시간 정보 , 유효/무효/피상전력량,
    상태정보
  • 정기LP : 1분, 5분, 10분, 15분, 30분, 60분
  • 비정기LP : 아래 항목 외에는 정기 LP로 기록
    • 정·복전 발생 : 정전시간으로 기록, 정기
      /비정기/계절 변경 계량값 확정시
    • 시간변경발생 : 시간변경전 시간을 기록
    • LP기록주기 변경:기록주기 변경 시간을
      기록
  • ※정기 비정기 LP를 발생순으로 저장해야한다.
@PUSH
  •  DLMS Data Notification Service사용하여야함.
@소프트웨어 업데이트
  • DLMS BlueBookEd 12.2.4.4.6 Image transfer를
    따라야 한다.
  • 원격 또는 현장에서 계기 계량부 및 운영 부
    소프트웨어를 업데이트할 수 있어야 한다.
@TOU 이어받기
  • DLMS BlueBookEd 12.2.4.4.6 Image transfer를
    따라야 한다.
@부하제한기
  • 계기는 상위 시스템에서 부하를 제어할 수 있는
    기능을 지원해야 한다.
@위상 거미
  • 위상검출 신호처리 기능을 지원해야한다.
@타임스위치 제어기능
  • 계기는 상위 시스템에서 타임스위치를 제어할
    수 있는 기능을 지원하여야 한다.
@전원보상
  • 무전원 시 배터리를 이용하여 시간정보를
    유지하여야한다.
  • 배터리 이용시 계량자료에 손실이없어야한다.
@계측계량정보
  • LCD에 표시된 내용과 저장정보 간의 상호
    무결성을 보장하여야 한다.
